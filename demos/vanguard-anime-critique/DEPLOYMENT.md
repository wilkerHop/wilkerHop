## Quick Start

Follow these steps to deploy your neo-brutalist Vanguard Anime Critique website to GitHub Pages.

### Step 1: Create GitHub Repository

1. Go to https://github.com/new
2. Fill in:
   - **Repository name**: `vanguard-anime-critique` (or your preferred name)
   - **Description**: "Neo-Brutalist Vanguard Anime Critique - A methodology for anime franchise discourse"
   - Visibility: **Public**
3. **Important**: Do NOT check:
   - ❌ Add a README file
   - ❌ Add .gitignore
   - ❌ Choose a license
4. Click **Create repository**

### Step 2: Push Your Code

After creating the repository, run these commands:

```bash
cd /Users/wilkerribeiro/.gemini/antigravity/playground/dynamic-belt

# Add your GitHub repository as remote
git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPO_NAME.git

# Push to GitHub
git push -u origin main
```

**Replace**:
- `YOUR_USERNAME` with your GitHub username
- `YOUR_REPO_NAME` with your repository name (e.g., `vanguard-anime-critique`)

### Step 3: Update Status Badge

Edit `README.md` and replace the badge URLs:

```markdown
[![Deploy to GitHub Pages](https://github.com/YOUR_USERNAME/YOUR_REPO_NAME/actions/workflows/deploy.yml/badge.svg)](https://github.com/YOUR_USERNAME/YOUR_REPO_NAME/actions/workflows/deploy.yml)
```

Replace `YOUR_USERNAME` and `YOUR_REPO_NAME` with your actual values, then commit:

```bash
git add README.md
git commit -m "Update status badge URL"
git push
```

### Step 4: Enable GitHub Pages

1. Go to your repository on GitHub
2. Click **Settings** (top menu bar)
3. Click **Pages** (left sidebar)
4. Under **Build and deployment**:
   - **Source**: Select **"GitHub Actions"**
5. Save (if required)

### Step 5: Wait for Deployment

1. Go to the **Actions** tab in your repository
2. You should see a workflow run called "Deploy to GitHub Pages"
3. Wait for the green checkmark (usually 1-2 minutes)
4. Your site will be live at: `https://YOUR_USERNAME.github.io/YOUR_REPO_NAME/`

## Status Badge Information

The status badge shows:
- **Passing** (green) - Website successfully deployed
- **Failing** (red) - Deployment error
- **In Progress** (yellow) - Currently deploydeploy

The badge is automatically generated by GitHub Actions and updates with each deployment.

## Troubleshooting

### Badge Not Showing
- Ensure you've replaced `YOUR_USERNAME` and `YOUR_REPO_NAME` in the badge URL
- Check that your repository is public
- Verify the workflow file is at `.github/workflows/deploy.yml`

### Deployment Failing
- Check the **Actions** tab for error logs
- Ensure GitHub Pages is enabled in Settings → Pages
- Verify Source is set to "GitHub Actions"

### Site Not Loading
- Wait a few minutes after first deployment
- Check Settings → Pages for the published URL
- Clear your browser cache
